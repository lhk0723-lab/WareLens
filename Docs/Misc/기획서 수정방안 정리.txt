MediaPipe

- 입력 데이터에 성별 추가

- 골반 비율 분석 제거
  (상의 중심 MVP이므로 추후 개발 항목으로 이동)

- 사이즈 기준을 자체 규칙이 아닌 KS 표준 사이즈표 기준으로 변경

- 사이즈 스펙 설명을 KS 표준 사이즈표 사용으로 수정

- 사이즈 판단 방법 명확화
  MediaPipe 랜드마크 기반 어깨 너비 및 상체 비율 추정
  + 키, 몸무게, BMI
  + KS 표준 사이즈표
  → 추천 사이즈 산출

- 핏 스타일(SLIM, OVER 등) 추천 기능 제거
  (추후 개발 항목으로 이동)

- '최종 사이즈' 표현 제거
  → '추천 사이즈'로 통일

- API 응답에 추천 근거(reason) 추가

- 개발 일정의 '모델 정확도 테스트' 표현 수정
  → 체형 분석 결과 검증
  → 사이즈 추천 로직 검증
  → 예외 상황 테스트




Backend
- 버전상 안정성 때문에 최신버전보다 Java 21 LTS 사용
- 서버간 통신도 WebClient로 확정


Integration

- CLIP ↔ MediaPipe 통합 설명 수정
  현재:
  MediaPipe 결과를 CLIP 입력으로 사용

  수정:
  CLIP 추천 결과와 MediaPipe 사이즈 추천 결과를 통합하여 최종 응답 제공

- A/B 테스트 관련 내용 삭제
  (공모전 MVP 범위 초과)

- Hit Rate, NDCG 기반 평가 내용 삭제 또는 향후 확장 계획으로 이동
  (Ground Truth 데이터셋이 없는 현재 구조에서는 측정 어려움)

- Rate Limiting 관련 내용 삭제 또는 향후 확장 계획으로 이동
  (운영 단계 이슈)

Frontend

- 추천 성향 선택(균형형 / 색상 중심 / 스타일 중심)
  → 향후 확장 기능으로 이동

  이유:
  현재 CLIP 추천 엔진에도 아직 구현 확정이 안 되어 있음

- 취향 분석 결과 표현 수정

  현재:
  취향 분석 결과
  선호 색상
  추천 스타일

  수정:
  업로드 이미지 기반 유사 스타일 분석
  추천 의류 결과
  추천 근거 정보

  이유:
  CLIP은 취향을 "분석"하는 모델이 아니라
  유사도를 계산하는 모델

- 체형 분석 결과 표현 수정

  현재:
  체형 분석 결과
  추천 사이즈
  추천 Fit

  수정:
  체형 분석 결과
  BMI 정보
  추천 사이즈
  추천 근거

  이유:
  Fit 추천은 현재 MVP 범위에서 제외

- 페이지 구조 단순화 검토

  현재:
  Home
  Profile
  Analysis
  Recommend

  추천:
  Home
  Upload
  Result

  또는

  Home
  Result

  이유:
  공모전 MVP 기준으로 화면이 너무 많음

- Result 페이지에 설명 카드 추가

  예시:

  취향 이미지 3장 분석 완료

  추천 결과:
  후드티 6개
  맨투맨 3개
  니트 1개

  추천 사이즈:
  L

  추천 근거:
  - 업로드 이미지와 높은 유사도
  - 어깨 비율 기준 L 추천

  이유:
  AI가 왜 추천했는지 설명 가능

  유사도 점수
